前言

星星是一个很神奇的东西，它的很多机制需要大量游玩才能摸透。本文将会讲述一些关于星星的细节知识，供读者快速熟悉。想到哪写到哪，没有什么顺序。

参考文献：Minepig233 大佬写的这篇专栏。

前置知识

判定区图

基本类型星星表 by TRG

Fes 星星指的是由多种基本类型星星连接成的星星，这种机制是 Festival 代加入的。

跳区

一句话概括：一条星星未划完部分的 第二个判定区 处于激活状态时，第一个判定区会被直接消除。

例如：

有一条 1-5 的星星。

  1. 按住 B1
    • 按下的那一刻会消除 A1
    • B1 不会被消除（被激活的区域松手时才会消除）
  2. 点击 C 区
    • 按下的那一刻会消除 B1（被跳过）
    • 松开的那一刻会消除 C
  3. 按住 A5
    • 按下的那一刻会消除 B5+A5（A5是最后一个判定区，不会等到松手时再消除）
参与星星判定的区域

事实上，为了与旧框判定保持一致，只有 ABC 区会参与星星的判定。（WiFi 星星较为特殊，后面再说）

所以跳区，以及所有与星星相关的讨论，提到的判定区都是 ABC 区。

跳区保护

本段做了适当简化

3 个区长度的星星（比如1-3，1>3），它的第二个判定区不能跳过。

对于 V 星星，它的两段分别计算长度；而 Fes 星星计算的是总长度，所以 Fes 星星一般都能跳区。

比如：

有一条 1V36 的星星。

  1. 点击 A1，A1 会被消除
  2. 点击 A3，没有反应（A2 被跳区保护）
  3. 点击 A2，A2 会被消除
  4. 点击 B4 再点击 A6，星星被完成（跳过了 A3 和 B5）

WiFi 星星的判定机制

以 1w5 为例：

它在判定上被拆解为 1-4，1-5 和 1-6，当三条星星都被完成时，WiFi被完成。

除此以外，为了防止WiFi太容易留尾，它的最后 3 个判定区判定被放宽了，详见基本类型星星表 by TRG。

因此 WiFi 是唯一一种有 D 区参与判定的星星。

留尾什么时候判绿

留尾一个判定区会判绿 Good ，两个区及以上会判灰 Miss 。
